Reading: 2021-USENIX-Phishpedia: A Hybrid Deep Learning Based Approach to Visually Identify Phishing Webpages

Code

Available

Research Question

To address two prominent technical challenges in phishing identification, i.e., (i) accurate recognition of identity logos on webpage screenshots, and (ii) matching logo variants of the same brands.

How

Dividing the phishing identification problem into 1) identify logo recognition [use the object detection] and 2) brand recognition problem [use Siamese model].

  • Object detection: generates a set of bounding boxes to annotate the position and size of the objects on the image. Using the Fast RCNN model. Input the screenshot, output the (i) predict the object class (i.e., logo or input box) and (ii) refine shape and size of each object.
  • Brand recognition problem: target brand list, maintain a list of logo variants and a list of legitimate domains. For logo comparison, they use Siamese neural network model (for the Siamese model, input conventional training procedure for Siamese model selects three images (I1, I2, I3) as a sample to calculate Triplet losswhere I1 and I2 belong to same class, and I3 belongs to another class. The goal of training is to make sure the similarity of images in the same class (sim(I1, I2)) should be larger than that in different classes (sim(I2, I3 ))., there are some related papers in this area !!!). Reading: 2021-USENIX-Inferring Phishing Intention via Webpage Appearance and Dynamics a Deep Vision Based Approach

Code

Available

I. Introduction

1. Situation of anti-phishing method

  1. Manually verified anti-phishing blacklist: use dynamic blacklists to track reported phishing URLs. eg. Google
  2. Classification-based methods: trained from collecting phishing dataset, perform binary classification
  3. Reference-based methods: detecting phishing attacks by referencing a set of representations of well-known brands.

2. Challenges

  1. The blacklists and collected phishing webpages used for training become obsolete very quickly, resulting in a never-ending cat-and-mouse game between phishing attacks and defenses that are based on blacklists/classification.
  2. Reference-based phishing detection solutions are developed based on the intuition that, the key goal of a phishing attack is to deceive users with pages visually similar to legitimate websites. Recent references are logo and screenshots, but screenshots as a reference do not perfectly capture brand intention, and logos as a reference miss to capture credential-taking intention.

3. Main Contribution

They propose a referenced-based phishing detection system that captures both brand intention and credential-taking intention.

II. PhishIntention: Design and Development

Given URL, screenshots, html code, a list of protected brand, logo match threshold, CRP search depth, output binary phishing result, target brand. For each brand, the authors maintain its logos and domains.

1. Technical Challenges:

  1. Layout Definition and Extraction. First, visually salient components are hard to be fully formalized. Second, extracting a layout is technically challenging. Heuristics for visually salient HTML elements are hard to define.
  2. Text-Logo Recognition. First, the logos under the same brand can be very diverse. Second, the logos (especially text logo) under different brands can be visually similar.
  3. Identifying and Confirming Credential-Taking Intention.

2. Steps

Step 1: Abstract Webpage Layout Detection (need labeled data).

  1. Definition of AWL: given a set of screenshots, AWL is a set of boxes that representing visually salient rectangular regions on screenshots.
  2. Process: construct a layout dataset of 9K webpage screenshots, annotated with regions and types of salient UI elements. Trained an object detection model (Fast R-CNN model) to predict salient UI elements on the screenshots.

Step 2: OCR-aided Brand Recognition.

  1. Challenge: Logo can be icon-like or text like image or both.
  2. Process: The autors applied OCR-adied Siamese matching model. Using 2 branches to get embedding of logos. One branch is ResNetV2-50 feature extractor to get appearance-based embedding, the other branch is OCR to get text-based embedding. Then task ASTER as model. Compare 2 logos’ cosine similarity.

Step 3: CRP Classification.

  1. Process: If match a brand, then build a CRP classifier taking the screenshot and the AWL as input, classifies whether the webpage requires user credentials. If is CRP but domain is not aligned with list brands’ domain, then it is phishing. If not, then next step

Step 4: CRP-Transition Location.

  1. Process: Search for button link to the CRP. So construct a dataset annotated by the CRP-button. Continue aforementioned process

I. Introduction

1. Situation of anti-phishing method

Phishing webpages try to deceive paper by high visual similarity between phishing webpage and legistimate ones. Some methods maintain a list of commonly attack pages (domain name and their screenshots). However, these methods have limitations: 1) list is small; 2) attackers can bypass detection by crafting partially similar phishing pages with different background pictures.

2. Contributions of this paper

  1. Present dataset of VisualPhish, 155 trust websites with 9363 pages.
  2. Present VisualPhishNet, a similarity-based detection model that is the first to utilize a deep learning approach (in particular, triplet convolutional neural networks) to learn a robust visual similarity metric between any two same-website webpages’ screenshots, instead of relying on one-to-one matching, which outperforms prior work by a large margin.

1. Paper summary

The authors proposed an anti-phishing method to protect Internet users from the phishing attacks. Specifically, they first obtained screenshots, then segmented them to get the logos. After that, they utilized Google image database to search and return keywords. Next, they used the keywords to search again, then compare domain name between the query website and the second search domain name.

2. Background

  • Earth Movers Distance (EMD): Fu et al. proposed a visual similarity method which uses EMD to measure the webpage visual similarity in distinguishing the phishing websites. They first converted the involved webpages into low resolution images and then used color and coordinate features to represent the image signatures. After that, they used EMD to calculate the signature distances of the converted images. If the EMD-based visual similarity of a webpage exceeds the threshold of a protected webpage, that page will be classified as a phishing websites. Since this method is assuming the phishing website is visually very similar to the targeted legitimate website, phishers can easily thwart the detection by decreasing slightly the degree of visual similarity.

3. Model

  • Main objective: phishing website try to deceive the users into believing that the phishing website they are visiting is the legitimate website
  • The model includes logo segmentation and website identity identification processes. In logo segmentation process, the logo is extracted from the website in query. Whereas in website identity identification process, extracted logo will be used to determine the identity of the website. In the second step, we use google database of different websites logos.
  • Whole Process: capture screenshots, segment, search, return keywords, search, compare domain name

Learning icons appearance similarity

What:

  1. main goal is to obtain a similarity metric between a pair of icons
  2. Present an icon dataset labeled by designers where each collection shares a coherent style and visual identity.
  3. We learn icons’ appearance similarity using a Siamese Neural Network with a triplet loss function and adaptive sampling trained from our weakly-labeled dataset and evaluated with human ratings.
  4. We propose several applications including search by similarity and a method to create icon sets optimized for style and visual identity in order to help users on user-interface design tasks.
  5. We collect annotated ratings on the perception of appearance similarity for iconography
  6. Sum: Presented a model for measuring the properties of style and visual identity in iconography. Specifically, trained a Siamese Neural Network with an on-line dataset of icons organized in visually coherent collections that are used to adaptively sample training data and optimize the training process. As the dataset contains noise, we further collect human-rated information on the perception of icon’s similarity which will be used for evaluating and testing the proposed model
  7. Our learned model maps each icon into a 256-dimensional feature space which allows direct comparisons by computing Euclidean distances How
  8. Our main goal is to obtain a similarity metric between a pair of icons
