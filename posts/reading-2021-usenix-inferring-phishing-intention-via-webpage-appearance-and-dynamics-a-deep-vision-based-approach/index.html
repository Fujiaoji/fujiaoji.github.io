I. Introduction

1. Situation of anti-phishing method

  1. Manually verified anti-phishing blacklist: use dynamic blacklists to track reported phishing URLs. eg. Google
  2. Classification-based methods: trained from collecting phishing dataset, perform binary classification
  3. Reference-based methods: detecting phishing attacks by referencing a set of representations of well-known brands.

2. Challenges

  1. The blacklists and collected phishing webpages used for training become obsolete very quickly, resulting in a never-ending cat-and-mouse game between phishing attacks and defenses that are based on blacklists/classification.
  2. Reference-based phishing detection solutions are developed based on the intuition that, the key goal of a phishing attack is to deceive users with pages visually similar to legitimate websites. Recent references are logo and screenshots, but screenshots as a reference do not perfectly capture brand intention, and logos as a reference miss to capture credential-taking intention.

3. Main Contribution

They propose a referenced-based phishing detection system that captures both brand intention and credential-taking intention.

II. PhishIntention: Design and Development

Given URL, screenshots, html code, a list of protected brand, logo match threshold, CRP search depth, output binary phishing result, target brand. For each brand, the authors maintain its logos and domains.

1. Technical Challenges:

  1. Layout Definition and Extraction. First, visually salient components are hard to be fully formalized. Second, extracting a layout is technically challenging. Heuristics for visually salient HTML elements are hard to define.
  2. Text-Logo Recognition. First, the logos under the same brand can be very diverse. Second, the logos (especially text logo) under different brands can be visually similar.
  3. Identifying and Confirming Credential-Taking Intention.

2. Steps

Step 1: Abstract Webpage Layout Detection (need labeled data).

  1. Definition of AWL: given a set of screenshots, AWL is a set of boxes that representing visually salient rectangular regions on screenshots.
  2. Process: construct a layout dataset of 9K webpage screenshots, annotated with regions and types of salient UI elements. Trained an object detection model (Fast R-CNN model) to predict salient UI elements on the screenshots.

Step 2: OCR-aided Brand Recognition.

  1. Challenge: Logo can be icon-like or text like image or both.
  2. Process: The autors applied OCR-adied Siamese matching model. Using 2 branches to get embedding of logos. One branch is ResNetV2-50 feature extractor to get appearance-based embedding, the other branch is OCR to get text-based embedding. Then task ASTER as model. Compare 2 logos’ cosine similarity.

Step 3: CRP Classification.

  1. Process: If match a brand, then build a CRP classifier taking the screenshot and the AWL as input, classifies whether the webpage requires user credentials. If is CRP but domain is not aligned with list brands’ domain, then it is phishing. If not, then next step

Step 4: CRP-Transition Location.
