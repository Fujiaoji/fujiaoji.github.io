3. Blacklist Evaluation metrics

3.1 Blacklist Performance

2 metrics:

  • Discovery refers to a blacklist’s ability to identify new URLs in the wild that are suspected of hosting phishing content.
  • Detection refers to a blacklist’s ability to correctly classify the discovered URLs, such that URLs with phishing content are added to the blacklist.
    • For any set of phishing URLs discovered by a blacklist, coverage is the proportion of these URLs that are blacklisted at any point while they host phishing content.
    • Speed is the time delay between discovery and blacklisting, which assesses how quickly blacklists respond.

3.2 Selection of Blacklists

  • Google Safe Browsing (GSB) protects Chrome, Safari, Firefox, and Chromium. Link
  • Microsoft SmartScreen protects Internet Explorer (IE) and Edge
  • Opera’s fraud and malware protection

4. PhishTime Overview

An effective way to empirically evaluate the performance of anti-phishing blacklists is to deploy a large batch of specially-configured test phishing websites, report the websites directly to blacklists, and then monitor each website to see if and when it is blacklisted.

The workflow:

  • Monitor Blacklisting of Live PhishingWebsites
  • Report URLs and Prune if Blacklisted.
  • Analyze (Evasive) PhishingWebsites: 手动分析为什么他们可以逃过blacklist detection: tech they use, appearance, user interface
    • 具体来说，visiting each URL, test different variations of request parameters until successfully retrive the content. Thus infer the server-side evasion tech used by phsihing website.
    • analyze each web visually, inspect the client-side source code
  • Design experiment (evasion inspirred)
    • After analyzing a representative sample of URLs, we abstract key trends anad design experiment to replicate them
  • Deploy Phishtime experiment
  • Ecosystem Recommendation
